# JavaScript APIs

Mystral Native.js implements standard Web APIs for graphics, audio, networking, and input.

## Supported APIs

| API | Status | Notes |
|-----|--------|-------|
| WebGPU | Full | Core graphics API via Dawn or wgpu-native |
| Canvas 2D | Full | Skia-based implementation |
| Web Audio | Full | SDL3-based audio |
| fetch | Full | HTTP/HTTPS and file:// protocols |
| WebTransport | Full | HTTP/3 (QUIC) via quiche — datagrams + reliable streams |
| WebSockets | In progress | Planned — see [Roadmap](#roadmap-realtime-networking) |
| WebRTC | In progress | Planned — see [Roadmap](#roadmap-realtime-networking) |
| URL / URLSearchParams | Full | URL parsing and query string handling |
| Worker | Full | Native OS thread with an independent JS engine |
| WorkerPool / transferable ArrayBuffer | Full | Deterministic barriers and binary ownership transfer |
| SharedBuffer / SharedTable / SharedQueue | Full | Zero-copy shared game state via `mystral/shared` |
| Blob | Full | Binary data containers |
| TextEncoder / TextDecoder | Full | UTF-8 encoding/decoding |
| Gamepad | Full | Controller support |
| requestAnimationFrame | Full | Frame timing |
| setTimeout/setInterval | Full | Timing functions |
| ES Modules | Full | import/export |
| TypeScript | Full | Via SWC transpilation |
| GLTF/GLB Loading | Full | Including Draco mesh compression |

## WebGPU

Full WebGPU API implementation via Dawn or wgpu-native.

```javascript
// Request adapter
const adapter = await navigator.gpu.requestAdapter({
    powerPreference: "high-performance"
});

// Request device
const device = await adapter.requestDevice();

// Get canvas context
const context = canvas.getContext("webgpu");
const format = navigator.gpu.getPreferredCanvasFormat();

context.configure({
    device,
    format,
    alphaMode: "premultiplied"
});

// Create shaders, pipelines, buffers, etc.
const shaderModule = device.createShaderModule({
    code: `
        @vertex fn vs() -> @builtin(position) vec4f { ... }
        @fragment fn fs() -> @location(0) vec4f { ... }
    `
});
```

## Canvas 2D

Skia-based 2D rendering context.

```javascript
const ctx = canvas.getContext("2d");

// Drawing operations
ctx.fillStyle = "#ff0000";
ctx.fillRect(10, 10, 100, 100);

ctx.strokeStyle = "#0000ff";
ctx.lineWidth = 2;
ctx.strokeRect(50, 50, 100, 100);

// Text
ctx.font = "24px sans-serif";
ctx.fillText("Hello World", 100, 100);

// Paths
ctx.beginPath();
ctx.arc(150, 150, 50, 0, Math.PI * 2);
ctx.fill();
```

## Web Audio

Full Web Audio API for sound playback and synthesis.

```javascript
const audioContext = new AudioContext();

// Load and decode audio
const response = await fetch("file://./sounds/music.mp3");
const arrayBuffer = await response.arrayBuffer();
const audioBuffer = await audioContext.decodeAudioData(arrayBuffer);

// Play audio
const source = audioContext.createBufferSource();
source.buffer = audioBuffer;
source.connect(audioContext.destination);
source.start();

// Audio processing
const gainNode = audioContext.createGain();
gainNode.gain.value = 0.5;
source.connect(gainNode);
gainNode.connect(audioContext.destination);
```

## fetch

HTTP/HTTPS requests and local file access.

```javascript
// Local files (relative to script)
const model = await fetch("file://./assets/model.glb");
const buffer = await model.arrayBuffer();

// Local files (absolute path)
const config = await fetch("file:///home/user/config.json");
const json = await config.json();

// HTTP/HTTPS
const response = await fetch("https://api.example.com/data");
const data = await response.json();

// POST request
const result = await fetch("https://api.example.com/upload", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ key: "value" })
});
```

## WebTransport

Low-latency client networking over HTTP/3 (QUIC), implemented natively via
[Cloudflare quiche](https://github.com/cloudflare/quiche). Supports unreliable
**datagrams** (UDP-like, for fast state updates) and reliable **streams**
(ordered, for chat, RPC, asset transfer) — the same `WebTransport` API you use in
the browser.

```javascript
const transport = new WebTransport("https://127.0.0.1:4433/echo");
await transport.ready;

// Unreliable datagrams
const writer = transport.datagrams.writable.getWriter();
await writer.write(new Uint8Array([1, 2, 3]));
const reader = transport.datagrams.readable.getReader();
const { value } = await reader.read(); // Uint8Array

// Reliable bidirectional stream
const stream = await transport.createBidirectionalStream();
const sw = stream.writable.getWriter();
await sw.write(new TextEncoder().encode("hello"));
await sw.close();
const sr = stream.readable.getReader();
const chunk = await sr.read(); // { value, done }

// Reliable unidirectional stream
const uni = await transport.createUnidirectionalStream();
await uni.getWriter().write(new Uint8Array([4, 5, 6]));

// Server-initiated streams
const incoming = transport.incomingUnidirectionalStreams.getReader();
const { value: serverStream } = await incoming.read();
```

Supported: `ready`/`closed`/`close()`, `datagrams`,
`createBidirectionalStream()`, `createUnidirectionalStream()`,
`incomingUnidirectionalStreams`, `incomingBidirectionalStreams`, and
`WebTransportError`.

> **Note:** WebTransport requires a build with quiche compiled in
> (`MYSTRAL_HAS_QUICHE`). The server URL must use `https://` with an explicit
> port. `serverCertificateHashes` (self-signed pinning) is not yet supported.
> See the [WebTransport guide](/mystralnative/docs/guides/webtransport) for full
> details, the test fixture, and architecture notes.

## Roadmap: Realtime networking

WebTransport is the first of a planned set of realtime networking APIs exposed
through the JS layer. Tracked in
[issue #17](https://github.com/mystralengine/mystralnative/issues/17):

| API | Status |
|-----|--------|
| WebTransport | ✅ Done — datagrams + reliable uni/bidi streams over HTTP/3 |
| WebSockets (`WebSocket`) | 🚧 In progress — reliable ordered messaging over TCP/TLS; likely libuv TCP + WebSocket framing |
| WebRTC (`RTCPeerConnection`, `RTCDataChannel`) | 🚧 In progress — P2P data channels for player-hosted/mesh networking + NAT traversal (ICE/STUN/TURN); evaluating `webrtc-rs` / libdatachannel |
| WebTransport `serverCertificateHashes` | 🚧 In progress — connect to self-signed servers via cert pinning |

## Gamepad

Standard Gamepad API for controller input.

```javascript
// Connection events
window.addEventListener("gamepadconnected", (e) => {
    console.log("Connected:", e.gamepad.id);
});

window.addEventListener("gamepaddisconnected", (e) => {
    console.log("Disconnected:", e.gamepad.id);
});

// Poll gamepad state in game loop
function update() {
    const gamepads = navigator.getGamepads();
    for (const gamepad of gamepads) {
        if (!gamepad) continue;

        // Axes (sticks)
        const leftX = gamepad.axes[0];  // -1 to 1
        const leftY = gamepad.axes[1];
        const rightX = gamepad.axes[2];
        const rightY = gamepad.axes[3];

        // Buttons
        const a = gamepad.buttons[0].pressed;
        const b = gamepad.buttons[1].pressed;
        const x = gamepad.buttons[2].pressed;
        const y = gamepad.buttons[3].pressed;
    }
}
```

## Timing

Animation frame and timing functions.

```javascript
// Animation loop
function render(timestamp) {
    // timestamp is in milliseconds
    const deltaTime = timestamp - lastTime;
    lastTime = timestamp;

    // Update and render
    update(deltaTime);
    draw();

    requestAnimationFrame(render);
}
requestAnimationFrame(render);

// Delayed execution
setTimeout(() => {
    console.log("Delayed");
}, 1000);

// Repeated execution
const intervalId = setInterval(() => {
    console.log("Tick");
}, 100);

// Cancel
clearInterval(intervalId);
```

## URL

Standard URL parsing and blob URL support.

```javascript
// Parse URLs
const url = new URL("https://example.com/path?key=value#hash");
console.log(url.hostname); // "example.com"
console.log(url.pathname); // "/path"

// Query parameters
const params = new URLSearchParams("foo=1&bar=2");
params.get("foo"); // "1"

// Blob URLs (used internally by libraries like Draco decoder)
const blob = new Blob(["console.log('hello')"]);
const blobUrl = URL.createObjectURL(blob);
URL.revokeObjectURL(blobUrl);
```

## Worker

Each Worker runs on its own OS thread with an independent JavaScript engine and
heap. Module workers can load JavaScript or TypeScript through the runtime module
resolver. JSON-compatible control data uses `postMessage`; shared game state is
passed by handle without copying its backing memory.

```javascript
const worker = new Worker('./workers/simulation.ts', {
    type: 'module',
    name: 'simulation',
});

worker.onmessage = (e) => console.log("Result:", e.data);
worker.postMessage(21); // Result: 42
worker.terminate();
```

Blob workers and classic `importScripts()` remain supported. The worker loop is
event-driven and sleeps until a message or timer is due. Worker globals include
real `setTimeout`, `clearTimeout`, `setInterval`, `clearInterval`, and
`queueMicrotask`. Timers use a monotonic native clock and do not busy-poll.

Plain `ArrayBuffer` values can be transferred with ownership semantics. The
sender is detached when `postMessage` accepts the transfer list:

```javascript
const vertices = new ArrayBuffer(4 * 1024 * 1024);
worker.postMessage({ vertices }, [vertices]);
console.log(vertices.byteLength); // 0
```

The receiving isolate gets a normal `ArrayBuffer` at the same property. V8 moves
the backing store without copying.
Duplicate entries, non-`ArrayBuffer` entries, and detached buffers are rejected.

Input and output user-message queues are bounded to 1,024 messages and 16 MiB per Worker.
`postMessage()` throws `RangeError` when the input queue is full or a serialized
message plus transferred bytes exceed the byte limit. Worker-side `postMessage()` throws when its
output queue reaches the same limit. Use `SharedQueue` for sustained high-rate
game traffic instead of relying on an unbounded message backlog.

`terminate()` interrupts currently executing JavaScript on V8, so a
Worker stuck in a pure JavaScript loop cannot block normal runtime shutdown or
hot reload.

### Deterministic WorkerPool

`mystral/worker-pool` keeps a fixed set of Workers alive, divides a numeric range
into stable contiguous partitions, and resolves each round only after every
Worker reaches the barrier. Results are always returned in Worker-index order.

```javascript
// simulation-pool.worker.mjs
import { exposeWorkerTask } from 'mystral/worker-pool';
import { CharacterRows } from './CharacterRows.mjs';

exposeWorkerTask(({ data, begin, end }) => {
    const characters = CharacterRows.attach(data.characters);
    for (let row = begin; row < end; row++) {
        characters.birthDay[row] -= 1;
    }
    return { updated: end - begin };
});
```

```javascript
// main.mjs
import { WorkerPool } from 'mystral/worker-pool';

const pool = new WorkerPool('./simulation-pool.worker.mjs', {
    size: 4,
    name: 'simulation',
});

const results = await pool.run(
    { characters: characters.handle() },
    { length: characters.length },
);
pool.terminate();
```

Only one round can be active per pool. If the range is not divisible by the pool
size, lower Worker indexes receive one extra row. A task error rejects the round
and terminates the pool so partially completed state cannot be silently reused.

### Shared game state

Schemas are ordinary modules imported by the main isolate and every Worker that
uses them:

```javascript
// CharacterRows.mjs
import { SharedTable } from 'mystral/shared';

export const CharacterRows = SharedTable.define('game.characters/v1', {
    templateIndex: 'u16',
    settlementId: 'u32',
    householdRef: 'u32',
    birthDay: 'i32',
});
```

```javascript
// main.mjs
import { CharacterRows } from './CharacterRows.mjs';

const characters = CharacterRows.create({ capacity: 200_000 });
const worker = new Worker('./simulation.worker.mjs', { type: 'module' });
worker.postMessage({ characters: characters.handle() });
```

```javascript
// simulation.worker.mjs
import { CharacterRows } from './CharacterRows.mjs';

self.onmessage = ({ data }) => {
    const characters = CharacterRows.attach(data.characters);
    characters.birthDay[0] = -10_000;
};
```

`SharedTable` uses fixed-capacity struct-of-arrays storage. `attach()` validates
the schema id, layout hash, capacity, and shared header. `definition.resize()`
performs an explicit migration into a new backing store; the old table stays
valid until it is released:

```javascript
// Stop writers and wait for the current round first.
const larger = CharacterRows.resize(characters, { capacity: 400_000 });
// Distribute larger.handle() and wait for Worker acknowledgements.
characters.release();
characters = larger;
```

`pool.resizeTable(CharacterRows, characters, { capacity })` applies the same
migration and rejects it while that pool has an active round. It does not manage
Workers outside the pool, so the application must still establish a global safe
point before retiring the old table. Attached TypedArray views are never replaced
transparently.

`SharedQueue.define()` creates a bounded single-producer/single-consumer ring
buffer. Payload fields can be `u8`, `i8`, `u16`, `i16`, `u32`, `i32`, `f32`, or
`f64`. Queue publication uses `Atomics`, while rows assigned exclusively to one
Worker can use normal TypedArray writes.

### Tradeoffs

- JavaScript objects and closures are private to their isolate.
- `postMessage` does not yet implement the browser structured-clone algorithm;
  payloads must be JSON-compatible, except for Mystral shared handles and
  explicitly transferred `ArrayBuffer` values.
- Rendering and WebGPU remain on the main thread.
- External shared backing stores and forced execution interruption are available
  on V8. Blocking `Atomics.wait()` is not exposed; use Worker messages or
  `SharedQueue` publication instead.

Run the fixed shared-memory benchmark with `mystral run benchmarks/worker-shared.mjs`
and the ownership-transfer benchmark with
`mystral run benchmarks/worker-transfer.mjs --no-sdl`.
